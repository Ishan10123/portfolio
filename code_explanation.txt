/app → UI Layer (Frontend Rendering Engine)
/package.json → Dependency & Runtime Controller
/tsconfig.json → Type System Rules
/next.config.js → Framework Behavior Control

This is Next.js App Router architecture

app/ = our product UI, this is where our actual application lives
package.json = what our system depends on
tsconfig.json = how strict our code is
next.config.js = framework tuning layer
layout.tsx = GLOBAL WRAPPER, Wraps every page, Controls global UI like: background, fonts, navigation, theme



package.json → PROJECT BRAIN

What each part does:
scripts:
Command	Purpose
npm run dev	start dev server
build	production build
start	run production

dependencies:
Package	Role
next	framework
react	UI engine
react-dom	browser rendering

Without this file → project cannot run



Flow:
layout.tsx wraps → page.tsx renders inside {children}

page.tsx → injected into layout.tsx → final HTML → rendered

Browser → Next.js Server → React Components → HTML Response → Browser Render

1. Next.js starts
2. Reads /app/page.tsx
3. Wraps it with /app/layout.tsx
4. Applies config from:
   → tsconfig.json
   → next.config.js
5. Uses dependencies from package.json
6. Sends HTML to browser
7. React hydrates UI
